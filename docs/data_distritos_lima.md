## 3. Mapeo: lo que pediste vs lo que tengo

| Campo pedido | Estado | Columna fuente Infobras |
|---|---|---|
| **nombre** | ✅ completo | `Nombre de obra` |
| **fecha inicio** | ✅ completo (cuando la entidad lo reportó) | `Fecha de inicio de obra` |
| **reporte paralizadas** | ✅ completo: marca + causal + fecha + días | `Existe Paralización` + `Causal de paralización` + `Fecha de paralización` + `Número de dias paralizado` |
| **avances** | ✅ completo: físico real, físico programado, financiero | `Avance Físico Real Acumulado (%)` + `Avance Físico Programado Acumulado (%)` + `Porcentaje de ejecución financiera` |
| **ubicación** | ⚠️ solo texto/dirección, **sin lat/lon** | `Dirección o información de referencia` + `Tipo de ubicación` |

## 4. Gaps detectados y de dónde podrían venir

### 4.1 Coordenadas (lat/lon) — el gap más importante para el MVP

Infobras DataSet **no incluye lat/lon**. Solo `Dirección o información de referencia` (texto) y `Tipo de ubicación` (exacta/referencial). Esto bloquea el filtro 'obras a 500m/1km/3km' del MD maestro §6.1.

**Opciones para resolverlo:**

| Opción | Costo | Cobertura esperada | Notas |
|---|---|---|---|
| Nominatim (OSM, gratis) | 0 USD, 1 req/seg | 60-75% por baja calidad de direcciones | Rate-limit duro; ~15 min para las 870 obras |
| Google Geocoding API | ~$5/1000 calls | 85-95% | Requiere API key. ~$0.04 las 870 obras |
| Ficha individual Infobras (HTML) | Gratis, scrape | Algunas obras tienen mapa embebido | Hay que parsear coordenadas del HTML — ver script 03 |
| Infobras `Mapa` (geoserver interno) | Posiblemente bloqueado | Desconocido | El HTML hace referencia a `http://11.162.107.180:8070/geoserver` (IP interna, no accesible) |

Recomendación: **Nominatim para batch inicial + ficha Infobras para las paralizadas (39 obras) que requieren precisión**.

### 4.2 Causal de paralización vacía (~23% de las paralizadas)

9 de las 39 obras paralizadas tienen `Causal = NaN`. Esto es un gap del propio Infobras (la entidad reportó la paralización pero no la causal). No es scrapeable desde otra fuente: lo correcto es mostrar 'Causal no reportada por la entidad' en el UI.

### 4.3 Direcciones faltantes (6 obras)

Las obras INFOBRAS 83578, 132147, 144572, 160050, 169395, 537440 no tienen `Dirección o información de referencia`. Quedan con ubicación a nivel distrito solamente. Same gap que 4.2: depende de la entidad reportar.

### 4.4 Contratistas: vista de concentración ≤8 UIT (gap más grande del MD maestro)

Tengo `RUC` + `Nombre razón social` del contratista de cada obra. Lo que **falta** es la concentración de ese RUC en compras menores ≤8 UIT por entidad — eso vive en **OECE Pentaho** que requiere Playwright (ver `analisis_scraping.md` §4 y `estado_scrapers.md`). La API OCDS Perú (que sí tenemos) **no** incluye contratos ≤8 UIT por definición.

### 4.5 Cruce con OCDS por proceso

OCDS API ya está scrapeada (catálogo + 20 releases de muestra) pero no se ha cruzado con las obras Infobras. El campo cruce sería `Código Único de Inversión` o `RUC del contratista`. Pendiente de un script de join.
